## Part A: Creating Directories, Files, and Editing Text Files

## Creating Directories — `mkdir`

### Basic Usage

```bash
# Create a single directory
mkdir projects

# Create a directory with a specific path
mkdir /home/student/projects
```

### Common Options

```bash
# -p  Create parent directories as needed (no error if existing)
mkdir -p projects/src/main/java

# -v  Verbose output — prints a message for each directory created
mkdir -v new_folder
# Output: mkdir: created directory 'new_folder'

# Combine -p and -v
mkdir -pv projects/src/main/java
# Output:
# mkdir: created directory 'projects'
# mkdir: created directory 'projects/src'
# mkdir: created directory 'projects/src/main'
# mkdir: created directory 'projects/src/main/java'
```

### Creating Multiple Directories

```bash
# Create multiple directories at once
mkdir dir1 dir2 dir3

# Create multiple nested directory trees at once
mkdir -p project/{src,bin,docs,tests}

# Brace expansion with nested levels
mkdir -p project/{src/{main,test},bin,docs/{user,admin}}

# This creates:
# project/
# ├── src/
# │   ├── main/
# │   └── test/
# ├── bin/
# └── docs/
#     ├── user/
#     └── admin/
```

### Setting Permissions on Creation

```bash
# -m  Set permissions (mode) when creating
mkdir -m 755 public_html
mkdir -m 700 private_data
```

> **Teaching Note:** Explain that `-p` is one of the most useful flags because scripts often need to ensure a directory path exists without failing if it's already there. Without `-p`, `mkdir` will error if the directory exists or if a parent directory is missing.

## Creating Files

### Using `touch`

```bash
# Create a single empty file
touch notes.txt

# Create multiple files at once
touch file1.txt file2.txt file3.txt

# Create files using brace expansion
touch report_{jan,feb,mar,apr}.txt

# Create numbered files
touch log_{1..10}.txt
# Creates: log_1.txt log_2.txt ... log_10.txt

# Create files with zero-padded numbers
touch backup_{01..05}.log
# Creates: backup_01.log backup_02.log ... backup_05.log
```

> **Teaching Note:** `touch` is primarily used to update the access and modification timestamps on a file. If the file does not exist, it creates an empty file. This dual behavior is important to understand.

```bash
# Update the timestamp of an existing file (does NOT erase contents)
touch existing_file.txt

# Set a specific timestamp
touch -t 202601011200 file.txt
# Sets timestamp to Jan 1, 2026 at 12:00
```

### Using Redirection to Create Files

```bash
# Create an empty file (or truncate an existing one to zero bytes)
> newfile.txt

# Create a file with content
echo "Hello World" > greeting.txt

# Append to a file (creates it if it doesn't exist)
echo "Another line" >> greeting.txt

# Create a multi-line file using a here document
cat << EOF > config.txt
server=localhost
port=8080
debug=false
EOF
```

> **⚠️ Warning for students:** `>` will **overwrite** an existing file without warning. `>>` will **append**. This is one of the most common mistakes beginners make.

### Creating Multiple Files and Directories Together

```bash
# Create a project structure in one sequence
mkdir -p myapp/{src,tests,docs} && touch myapp/src/{main.py,utils.py} myapp/tests/test_main.py myapp/README.md

# Resulting structure:
# myapp/
# ├── src/
# │   ├── main.py
# │   └── utils.py
# ├── tests/
# │   └── test_main.py
# ├── docs/
# └── README.md
```

## Editing Files with `nano`

### Opening and Creating Files

```bash
# Open an existing file
nano myfile.txt

# Open a file and go to a specific line number
nano +15 myfile.txt

# Open a file in read-only mode
nano -v myfile.txt
```

### Key Commands Inside `nano`

| Shortcut       | Action                        |
|----------------|-------------------------------|
| `Ctrl + O`     | Save (Write Out)              |
| `Ctrl + X`     | Exit nano                     |
| `Ctrl + K`     | Cut the current line          |
| `Ctrl + U`     | Paste (Uncut) the cut line    |
| `Ctrl + W`     | Search for text               |
| `Ctrl + \`     | Search and Replace            |
| `Ctrl + G`     | Display Help                  |
| `Ctrl + C`     | Show cursor position (line #) |
| `Alt + U`      | Undo                          |
| `Alt + E`      | Redo                          |
| `Ctrl + A`     | Jump to beginning of line     |
| `Ctrl + E`     | Jump to end of line           |
| `Ctrl + _`     | Go to a specific line number  |

> **Teaching Note:** `nano` is beginner-friendly because all available commands are shown at the bottom of the screen. The `^` symbol means `Ctrl`. For example, `^X` means `Ctrl + X`.

## Editing Files with `vim`

### Opening Files

```bash
# Open a file (creates it if it doesn't exist)
vim myfile.txt

# Open a file at a specific line
vim +20 myfile.txt

# Open a file and search for a pattern
vim +/searchterm myfile.txt

# Open multiple files
vim file1.txt file2.txt
# Use :n to go to next file, :prev to go back

# Open a file in read-only mode
vim -R myfile.txt
```

### Understanding Vim Modes

Vim is a **modal editor**, meaning it operates in distinct modes. This is the single most important concept for students to understand.

| Mode       | Purpose                              | How to Enter                     |
|------------|--------------------------------------|----------------------------------|
| **Normal** | Navigate, delete, copy, paste        | Press `Esc` (default mode)       |
| **Insert** | Type and edit text                   | Press `i`, `a`, `o`, etc.        |
| **Visual** | Select text                          | Press `v`, `V`, or `Ctrl + v`    |
| **Command**| Execute commands (save, quit, etc.)  | Press `:` from Normal mode       |

> **Teaching Note:** The #1 mistake new Vim users make is trying to type text while in Normal mode. Always remind students: **press `i` to type, press `Esc` to stop typing.**

### Entering Insert Mode (Multiple Ways)

| Key | Action                                           |
|-----|--------------------------------------------------|
| `i` | Insert before the cursor                         |
| `I` | Insert at the beginning of the line              |
| `a` | Append after the cursor                          |
| `A` | Append at the end of the line                    |
| `o` | Open a new line below and enter Insert mode      |
| `O` | Open a new line above and enter Insert mode      |

### Saving and Quitting (Command Mode — press `:` first)

| Command   | Action                                        |
|-----------|-----------------------------------------------|
| `:w`      | Save (write) the file                         |
| `:q`      | Quit (fails if unsaved changes)               |
| `:wq`     | Save and quit                                 |
| `:x`      | Save and quit (same as `:wq`)                 |
| `ZZ`      | Save and quit (Normal mode shortcut)          |
| `:q!`     | Quit without saving (discard changes)         |
| `:w filename` | Save as a different filename              |
| `:wq!`    | Force save and quit                           |

### Navigation in Normal Mode

| Key             | Action                                  |
|-----------------|-----------------------------------------|
| `h`             | Move left                               |
| `j`             | Move down                               |
| `k`             | Move up                                 |
| `l`             | Move right                              |
| `w`             | Jump forward one word                   |
| `b`             | Jump backward one word                  |
| `0`             | Jump to beginning of line               |
| `$`             | Jump to end of line                     |
| `gg`            | Jump to first line of file              |
| `G`             | Jump to last line of file               |
| `5G` or `:5`    | Jump to line 5                          |
| `Ctrl + f`      | Page down (forward)                     |
| `Ctrl + b`      | Page up (backward)                      |
| `Ctrl + d`      | Half page down                          |
| `Ctrl + u`      | Half page up                            |

> **Teaching Note:** Arrow keys also work in Vim, but `h`, `j`, `k`, `l` are the "proper" Vim way and are much faster once learned. For this course, arrow keys are fine.

### Editing in Normal Mode

| Key       | Action                                          |
|-----------|-------------------------------------------------|
| `x`       | Delete the character under the cursor           |
| `dd`      | Delete (cut) the entire current line            |
| `5dd`     | Delete 5 lines                                  |
| `dw`      | Delete from cursor to end of word               |
| `d$`      | Delete from cursor to end of line               |
| `d0`      | Delete from cursor to beginning of line         |
| `yy`      | Yank (copy) the current line                    |
| `5yy`     | Yank 5 lines                                    |
| `yw`      | Yank from cursor to end of word                 |
| `p`       | Paste after the cursor                          |
| `P`       | Paste before the cursor                         |
| `u`       | Undo last change                                |
| `Ctrl + r`| Redo                                            |
| `.`       | Repeat the last command                         |
| `r`       | Replace the character under cursor (single char)|
| `cc`      | Change (replace) entire line                    |
| `cw`      | Change (replace) to end of word                 |

### Searching in Vim

| Command          | Action                                    |
|------------------|-------------------------------------------|
| `/searchterm`    | Search forward for "searchterm"           |
| `?searchterm`    | Search backward for "searchterm"          |
| `n`              | Go to next search match                   |
| `N`              | Go to previous search match               |
| `:%s/old/new/g`  | Replace all occurrences in the file       |
| `:%s/old/new/gc` | Replace all with confirmation             |
| `:s/old/new/g`   | Replace all occurrences on current line   |

### Visual Mode (Selecting Text)

| Key        | Action                                      |
|------------|---------------------------------------------|
| `v`        | Start character-wise visual selection        |
| `V`        | Start line-wise visual selection             |
| `Ctrl + v` | Start block (column) visual selection       |

Once text is selected:
- `d` — delete selection
- `y` — yank (copy) selection
- `>` — indent selection
- `<` — unindent selection

### Useful Vim Settings (Command Mode)

```
:set number          " Show line numbers
:set nonumber        " Hide line numbers
:set relativenumber  " Show relative line numbers
:set syntax on       " Enable syntax highlighting
:set tabstop=4       " Set tab width to 4 spaces
:set expandtab       " Use spaces instead of tabs
:set autoindent      " Auto-indent new lines
```

> **Teaching Note:** Walk students through a basic Vim workflow:
> 1. `vim filename.txt` — open or create the file
> 2. Press `i` — enter Insert mode
> 3. Type your text
> 4. Press `Esc` — return to Normal mode
> 5. Type `:wq` — save and quit
>
> Once they're comfortable with this, introduce navigation and editing commands gradually.

## The `ls` Command — Listing Directory Contents

### Basic Usage

```bash
# List files and directories in current directory
ls

# List contents of a specific directory
ls /etc

# List multiple directories
ls /etc /var
```

### Common Options

```bash
# -l  Long format (permissions, owner, size, date)
ls -l

# -a  Show ALL files, including hidden files (starting with .)
ls -a

# -la  Combine: long format + hidden files (most commonly used)
ls -la

# -lh  Long format with human-readable file sizes (KB, MB, GB)
ls -lh

# -lt  Long format, sorted by modification time (newest first)
ls -lt

# -ltr  Long format, sorted by time, reversed (oldest first)
ls -ltr

# -lS  Long format, sorted by file size (largest first)
ls -lS

# -R  Recursive — list contents of subdirectories
ls -R

# -d  List directories themselves, not their contents
ls -d */

# -1  List one file per line (useful for scripting)
ls -1

# -i  Show inode number
ls -i
```

### Reading `ls -l` Output

```
-rw-r--r-- 1 student student 4096 Mar  1 10:30 notes.txt
│├────────┤ │ │       │       │    │             │
│ │         │ │       │       │    │             └── Filename
│ │         │ │       │       │    └── Last modified date
│ │         │ │       │       └── File size (bytes)
│ │         │ │       └── Group owner
│ │         │ └── Owner
│ │         └── Hard link count
│ └── Permissions (rwxrwxrwx = owner/group/other)
└── File type (- = file, d = directory, l = symlink)
```

> **Teaching Note:** Breaking down the `ls -l` output is critical. Students need to be able to read permissions, ownership, and file types. Spend extra time on the permission bits (rwx) and what they mean for owner, group, and other.

### Advanced / Useful Combinations

```bash
# List only directories
ls -ld */

# List files sorted by extension
ls -lX

# List with file type indicators (/ for dirs, * for executables, @ for links)
ls -F

# List only hidden files
ls -ld .*

# Count the number of files in a directory
ls -1 | wc -l

# List files matching a pattern (glob)
ls *.txt
ls report_*.pdf
ls file?.txt          # ? matches a single character
ls file[1-3].txt      # matches file1.txt, file2.txt, file3.txt

# Show full timestamps
ls -l --full-time

# List files recursively with human-readable sizes
ls -lRh
```

## The `cat` Command — Concatenate and Display File Contents

### Basic Usage

```bash
# Display contents of a file
cat myfile.txt

# Display contents of multiple files
cat file1.txt file2.txt

# Concatenate files into a new file
cat file1.txt file2.txt > combined.txt

# Append a file's content to another file
cat extra.txt >> existing.txt
```

### Common Options

```bash
# -n  Number all output lines
cat -n myfile.txt

# -b  Number only non-blank lines
cat -b myfile.txt

# -s  Squeeze multiple blank lines into one
cat -s myfile.txt

# -E  Display $ at the end of each line (shows trailing spaces/line endings)
cat -E myfile.txt

# -T  Display TAB characters as ^I
cat -T myfile.txt

# -A  Show all non-printing characters (equivalent to -vET)
cat -A myfile.txt
```

### Creating Files with `cat`

```bash
# Create a file interactively (type content, then Ctrl+D to save)
cat > newfile.txt
This is line one.
This is line two.
# Press Ctrl+D to end input and save

# Append interactively
cat >> newfile.txt
This line will be added.
# Press Ctrl+D to end input

# Create a file with a here document
cat << EOF > script_config.txt
hostname=webserver01
ip=192.168.1.100
role=web
EOF
```

### Related Commands for Viewing Files

```bash
# tac — display file in reverse (last line first)
tac myfile.txt

# head — display first 10 lines (default)
head myfile.txt
head -n 5 myfile.txt       # first 5 lines
head -20 myfile.txt        # first 20 lines

# tail — display last 10 lines (default)
tail myfile.txt
tail -n 5 myfile.txt       # last 5 lines
tail -f /var/log/syslog    # follow — live updates (great for log files)

# less — paginated file viewer (use arrow keys, q to quit)
less myfile.txt

# more — older paginated viewer (space to advance, q to quit)
more myfile.txt
```

> **Teaching Note:** `cat` is one of the most frequently used commands. Emphasize:
> - `cat` is for **viewing** short files and **concatenating** files together.
> - For long files, use `less` instead — it lets you scroll.
> - `cat > file` creates/overwrites; `cat >> file` appends.
> - `Ctrl+D` signals end-of-input (EOF) when using `cat >` interactively.

## The `whereis` Command

`whereis` locates the **binary**, **source**, and **manual page** files for a command. It searches a predefined set of standard directories.

### Basic Usage

```bash
# Find the binary, source, and man page for a command
whereis ls
# Output: ls: /usr/bin/ls /usr/share/man/man1/ls.1.gz

whereis python3
# Output: python3: /usr/bin/python3 /usr/lib/python3 /usr/share/man/man1/python3.1.gz

whereis bash
# Output: bash: /usr/bin/bash /usr/share/man/man1/bash.1.gz
```

### Common Options

```bash
# -b  Search only for binaries
whereis -b ls
# Output: ls: /usr/bin/ls

# -m  Search only for manual pages
whereis -m ls
# Output: ls: /usr/share/man/man1/ls.1.gz

# -s  Search only for source files
whereis -s gcc

# -u  Find commands with unusual (missing) entries
#     Useful for finding commands that have a binary but no man page, etc.
whereis -u -m *
```

> **Teaching Note:** `whereis` is fast because it only searches standard system directories (like `/usr/bin`, `/usr/sbin`, `/usr/share/man`). It does NOT search your entire filesystem. It's useful for quickly finding where a system command lives and if documentation exists for it.

## The `find` Command

`find` is the most powerful file-searching command in Linux. It recursively searches directory trees based on a wide variety of criteria.

### Basic Syntax

```
find [starting_path] [options/tests] [actions]
```

### Search by Name

```bash
# Find a file by exact name (starting from current directory)
find . -name "notes.txt"

# Find a file by name (case-insensitive)
find . -iname "notes.txt"

# Find files by pattern (wildcard)
find /home -name "*.txt"

# Find files matching multiple patterns
find . -name "*.txt" -o -name "*.md"
# -o means OR
```

### Search by Type

```bash
# Find only files
find . -type f

# Find only directories
find . -type d

# Find only symbolic links
find . -type l

# Combine: find all .log files
find /var/log -type f -name "*.log"
```

### Search by Size

```bash
# Find files larger than 100MB
find / -type f -size +100M

# Find files smaller than 1KB
find . -type f -size -1k

# Find files exactly 50 bytes
find . -type f -size 50c

# Size suffixes: c=bytes, k=kilobytes, M=megabytes, G=gigabytes
```

### Search by Time

```bash
# Find files modified in the last 7 days
find . -type f -mtime -7

# Find files modified MORE than 30 days ago
find . -type f -mtime +30

# Find files accessed in the last 2 days
find . -type f -atime -2

# Find files changed in the last 60 minutes
find . -type f -mmin -60
```

> **Time Explanation:**
> - `-mtime` — modification time (in days)
> - `-atime` — access time (in days)
> - `-ctime` — status change time / metadata change (in days)
> - `-mmin`, `-amin`, `-cmin` — same as above but in **minutes**
> - `-7` means "less than 7 days ago" (within last 7 days)
> - `+30` means "more than 30 days ago"

### Search by Permissions

```bash
# Find files with exact permissions 777
find . -type f -perm 777

# Find files where owner has write permission (at minimum)
find . -type f -perm -u=w

# Find files writable by others (security audit)
find / -type f -perm -o=w

# Find all SUID files (important for security)
find / -type f -perm -4000
```

### Search by Owner

```bash
# Find files owned by a specific user
find /home -user student

# Find files owned by a specific group
find /home -group students

# Find files with no owner (orphaned files)
find / -nouser

# Find files with no group
find / -nogroup
```

### Search by Depth

```bash
# Limit search depth (don't recurse deeper than 2 levels)
find . -maxdepth 2 -name "*.txt"

# Start searching only from depth 2
find . -mindepth 2 -name "*.conf"
```

### Performing Actions on Results

```bash
# Print results (default action)
find . -name "*.txt" -print

# Delete found files (USE WITH CAUTION)
find /tmp -type f -name "*.tmp" -delete

# Execute a command on each result
find . -name "*.log" -exec cat {} \;
# {} is replaced by the filename, \; terminates the -exec

# Execute with confirmation (prompt before each)
find . -name "*.bak" -ok rm {} \;

# More efficient execution (batch mode)
find . -name "*.txt" -exec grep "error" {} +
# The + passes as many files as possible to a single grep command
```

### Practical Examples

```bash
# Find all empty files
find . -type f -empty

# Find all empty directories
find . -type d -empty

# Find duplicate filenames in different directories
find / -type f -name "config.yaml" 2>/dev/null

# Find large files (top disk consumers)
find / -type f -size +500M -exec ls -lh {} \; 2>/dev/null

# Find recently modified config files
find /etc -type f -name "*.conf" -mtime -1

# Find and count all Python files
find . -type f -name "*.py" | wc -l

# Find files and copy them somewhere
find . -name "*.jpg" -exec cp {} /backup/images/ \;

# Find all files NOT matching a pattern
find . -type f ! -name "*.txt"
```

> **Teaching Note:** `find` is one of the most important commands for system administration. Key points:
> - Always specify a starting path (`.` for current directory, `/` for entire system)
> - Searching from `/` can be slow — be more specific when possible
> - Use `2>/dev/null` to suppress "Permission denied" errors when searching system directories
> - The `-exec` action is extremely powerful — it lets you run any command on the results
> - **Always test with `-print` before using `-delete` or `-exec rm`**

## The `which` Command

`which` shows the **full path of a command's executable** by searching the directories listed in your `$PATH` environment variable.

### Basic Usage

```bash
# Find where a command's executable is
which ls
# Output: /usr/bin/ls

which python3
# Output: /usr/bin/python3

which nano
# Output: /usr/bin/nano
```

### Common Options

```bash
# -a  Show ALL matching executables in PATH (not just the first)
which -a python
# Output:
# /usr/bin/python
# /usr/local/bin/python
# This reveals if multiple versions exist in different PATH locations
```

### Practical Uses

```bash
# Check if a command is installed
which docker
# If nothing is returned, docker is not installed (or not in PATH)

# Find out which version of a command will run
which java
# Tells you the exact binary that runs when you type "java"

# Use in scripts to verify a command exists
if which curl > /dev/null 2>&1; then
    echo "curl is installed"
else
    echo "curl is NOT installed"
fi
```

> **Teaching Note:** `which` ONLY searches the `$PATH` — it finds executables that you can run by name. It will not find built-in shell commands (like `cd`), aliases, or functions. For that, use `type`.

## The `type` Command

`type` is a **shell built-in** that tells you how the shell interprets a command — whether it's a built-in, an alias, a function, or an external program.

### Basic Usage

```bash
# Check the type of a command
type ls
# Output: ls is aliased to `ls --color=auto`

type cd
# Output: cd is a shell builtin

type vim
# Output: vim is /usr/bin/vim

type if
# Output: if is a shell keyword
```

### Common Options

```bash
# -t  Print only the type (alias, builtin, file, function, keyword)
type -t ls
# Output: alias

type -t cd
# Output: builtin

type -t find
# Output: file

type -t for
# Output: keyword

# -a  Display ALL locations / definitions (including aliases and PATH entries)
type -a ls
# Output:
# ls is aliased to `ls --color=auto`
# ls is /usr/bin/ls

type -a echo
# Output:
# echo is a shell builtin
# echo is /usr/bin/echo

# -p  Show the disk file path (even if it's aliased)
type -p ls
# Output: /usr/bin/ls
```

### Types of Commands

| Type       | Description                                             | Example          |
|------------|---------------------------------------------------------|------------------|
| `alias`    | A shortcut defined in your shell config                 | `ls`, `ll`       |
| `builtin`  | Built into the shell itself, no external binary         | `cd`, `echo`, `pwd` |
| `file`     | An external executable program on disk                  | `vim`, `find`, `grep` |
| `function` | A shell function defined in your environment            | Custom functions |
| `keyword`  | A shell reserved word (part of shell syntax)            | `if`, `for`, `while` |

> **Teaching Note:** `type` is the most comprehensive command for determining what happens when you type a command name. It answers the question: *"When I type `ls`, what actually runs?"*
>
> This is important because:
> - An alias might override the default behavior (e.g., `ls` → `ls --color=auto`)
> - A built-in version might behave differently from an external version (`echo` has both)
> - Understanding command resolution order: **aliases → functions → builtins → PATH executables**

## Comparison: `which` vs `whereis` vs `type` vs `find`

| Command    | Searches          | Finds                          | Speed  | Use Case                                |
|------------|-------------------|--------------------------------|--------|-----------------------------------------|
| `which`    | `$PATH` only      | Executable path                | Fast   | "Where is this command's binary?"       |
| `whereis`  | Standard dirs     | Binary, source, man pages      | Fast   | "Where are all parts of this command?"  |
| `type`     | Shell environment  | Type and location of command   | Fast   | "What happens when I type this?"        |
| `find`     | Entire filesystem  | Any file based on criteria     | Slow*  | "Find files matching complex criteria"  |

*`find` traverses the filesystem in real-time, so it can be slow on large directory trees.

> **Teaching Note:** A good way to demonstrate the difference:
> ```bash
> which echo        # Shows: /usr/bin/echo
> whereis echo      # Shows: binary + man page locations
> type echo         # Shows: echo is a shell builtin (AND /usr/bin/echo)
> find / -name echo # Shows: every file named "echo" on the entire system
> ```
> Each gives you a different perspective on the same command.
