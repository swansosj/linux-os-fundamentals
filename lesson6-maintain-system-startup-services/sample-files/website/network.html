DNS & Networking

How domain names become IP addresses — powered by BIND9 running on this server.

What is DNS?

The Domain Name System (DNS) is the "phone book" of the internet. It translates human-readable domain names into machine-readable IP addresses.

You TypeDNS Resolves ToRecord Type
www.lab.local192.168.1.100A Record
ns1.lab.local192.168.1.100A Record
mail.lab.local192.168.1.100A Record
webserver.lab.local→ www.lab.localCNAME (alias)
lab.local (mail)→ mail.lab.localMX Record
192.168.1.100→ ns1.lab.localPTR (reverse)
All these records are defined in /etc/bind/zones/db.lab.local on this server. You can edit that file, increment the serial number, and run sudo rndc reload to add new records.

DNS Record Types Explained

A Record — Address

Maps a hostname directly to an IPv4 address. This is the most common record type.

www     IN      A       192.168.1.100
; "www.lab.local resolves to 192.168.1.100"

CNAME Record — Canonical Name (Alias)

Creates an alias that points to another hostname. Useful for multiple names for the same server.

webserver   IN      CNAME   www.lab.local.
; "webserver.lab.local is an alias for www.lab.local"

MX Record — Mail Exchange

Specifies which server handles email for the domain. The number (10) is priority — lower = preferred.

@       IN      MX  10  mail.lab.local.
; "Email for lab.local goes to mail.lab.local with priority 10"

NS Record — Name Server

Declares which server is authoritative for this domain's DNS zone.

@       IN      NS      ns1.lab.local.
; "ns1.lab.local is the name server for lab.local"

PTR Record — Pointer (Reverse DNS)

The opposite of an A record — maps an IP address back to a hostname.

100     IN      PTR     www.lab.local.
; "192.168.1.100 resolves back to www.lab.local"

SOA Record — Start of Authority

Contains metadata about the zone: primary name server, admin email, serial number, and timing values.

@   IN  SOA  ns1.lab.local. admin.lab.local. (
                2026030201  ; Serial — MUST increment on every change!
                3600        ; Refresh — how often secondaries check for updates
                1800        ; Retry — how long before retrying a failed refresh
                604800      ; Expire — when secondaries stop answering
                86400       ; Negative TTL — how long to cache "not found"
         )

Testing DNS — Commands to Try

dig (DNS Information Groper)

# Basic lookup
$ dig www.lab.local

# Short answer only
$ dig www.lab.local +short
192.168.1.100

# Query specific DNS server
$ dig @192.168.1.100 www.lab.local

# Reverse lookup
$ dig -x 192.168.1.100 +short
ns1.lab.local.

# Query MX records
$ dig lab.local MX

# Query all record types
$ dig lab.local ANY

nslookup

$ nslookup www.lab.local
$ nslookup www.lab.local 192.168.1.100

host

$ host www.lab.local
www.lab.local has address 192.168.1.100

The Zone File — Where Records Live

This is the actual zone file powering the DNS for lab.local:

; /etc/bind/zones/db.lab.local
$TTL    604800
@       IN      SOA     ns1.lab.local. admin.lab.local. (
                        2026030201      ; Serial
                        3600            ; Refresh
                        1800            ; Retry
                        604800          ; Expire
                        86400 )         ; Negative Cache TTL

@       IN      NS      ns1.lab.local.

ns1     IN      A       192.168.1.100
www     IN      A       192.168.1.100
web     IN      A       192.168.1.100
mail    IN      A       192.168.1.100
ftp     IN      A       192.168.1.100
db      IN      A       192.168.1.100
app     IN      A       192.168.1.100

webserver   IN  CNAME   www.lab.local.
website     IN  CNAME   www.lab.local.
dns         IN  CNAME   ns1.lab.local.

@       IN      MX  10  mail.lab.local.
@       IN      TXT     "v=spf1 mx -all"
