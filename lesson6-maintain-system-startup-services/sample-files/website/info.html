Inspecting HTTP Headers

HTTP headers are metadata sent between the browser and server. Here's how to see them:

View Response Headers with curl

# -I shows only headers (HEAD request)
$ curl -I http://www.lab.local
HTTP/1.1 200 OK
Server: nginx/1.22.1
Date: Sun, 01 Mar 2026 15:00:00 GMT
Content-Type: text/html
Content-Length: 5432
Last-Modified: Sun, 01 Mar 2026 10:00:00 GMT
Connection: keep-alive
ETag: "65e2a1b0-1538"
Accept-Ranges: bytes

View Request AND Response Headers

# -v (verbose) shows the full conversation
$ curl -v http://www.lab.local 2>&1 | head -25
*   Trying 192.168.1.100:80...
* Connected to www.lab.local (192.168.1.100) port 80
> GET / HTTP/1.1
> Host: www.lab.local
> User-Agent: curl/7.88.1
> Accept: */*
>
< HTTP/1.1 200 OK
< Server: nginx/1.22.1
< Content-Type: text/html
< Content-Length: 5432
...

Lines starting with > = what your client sent (request)
Lines starting with < = what the server replied (response)

Custom Headers on This Page

This /info.html page has custom headers added by the Nginx config:

$ curl -I http://www.lab.local/info.html
HTTP/1.1 200 OK
Server: nginx/1.22.1
X-Served-By: Nginx on Debian
X-Lab: FSCJ Linux Fundamentals
...

Those X- headers were added in the Nginx config:

location = /info.html {
    add_header X-Served-By "Nginx on Debian";
    add_header X-Lab "FSCJ Linux Fundamentals";
}

Quick Diagnostics

# Everything you need to troubleshoot in 60 seconds:

$ systemctl is-active nginx bind9          # Are they running?
$ sudo ss -tlnp | grep -E ':53|:80'        # Are ports open?
$ sudo nginx -t                             # Is the config valid?
$ sudo named-checkconf                      # Is DNS config valid?
$ dig @localhost www.lab.local +short       # Does DNS resolve?
$ curl -I http://localhost                  # Does HTTP respond?
$ sudo journalctl -u nginx -n 5            # Recent nginx logs?
$ sudo journalctl -u bind9 -n 5            # Recent DNS logs?
